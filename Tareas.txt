Requisitos funcionales
1. Registro de usuarios:
• Permitir que un nuevo usuario se registre proporcionando únicamente un nombre de
usuario y una contraseña. OK
• Informar al usuario si ya existe un registro con el mismo nombre de usuario. OK
• No permitir modificaciones a los datos de usuarios después del registro. OK
2. Inicio de sesión:
• Permitir a los usuarios registrados iniciar sesión introduciendo su nombre de usuario y
contraseña. OK
3. Verificar credenciales:
• Validar las credenciales proporcionadas contra los datos almacenados en el servidor. OK
• Denegar acceso si las credenciales no coinciden con un registro válido. OK
4. Cerrar sesión:
• Permitir a los usuarios ya logados cerrar la sesión. OK
5. Gestión de usuarios preexistentes:
• El sistema debe contar con un conjunto inicial de usuarios registrados para que
puedan acceder sin necesidad de registrarse.
6. Transacciones:
• Permitir a los usuarios autenticados realizar transacciones en el formato, no se debe
validar cuentas o cantidades:
o "Cuenta origen, Cuenta destino, Cantidad transferida". OK
7. Persistencia de datos:
• Registrar y almacenar de manera permanente:
o Los datos de los usuarios. OK
8. Interfaz de comunicación:
• Proveer una interfaz (usando sockets) que permita a los clientes interactuar con el
sistema de registro, autenticación, y transacciones. OK
